    《计算机科学中的随机过程》开源习题库安全守则

    欢迎来到本习题库。本仓库是《计算机科学中的随机过程》一书的配套习题。为了确保您在提交 PR 后依然能分清现实与状态空间，请背诵并严格遵守以下规则，请务必...&%))遵守*！

    绝对原则：如果你在访问本网站时发现屏幕上的字迹开始变得模糊且眼睛感到酸涩，请立刻闭上双眼，放缓呼吸，不要发出声音，保持两分钟后，你会变得安全且舒适。不要试图通过解答习题来脱困！

    第一部分：关于题目本身

    1. 每一章习题分为两部分：练习和问题。

    2. 练习部分的题目相对基础。我们尽最大努力确保且存在凡人可解的答案。请不要在深夜意识模糊的时候作答这部分。

    3. 问题部分更强调“综合能力”。在作答这部分时，如果某个离散状态空间的大小达到了 \(2^n\) 且毫无规律可言，请立刻放弃写出完整的状态转移矩阵。请耐心、真诚地祈祷秩序的降临。千万不要试图求出那个不存在的稳态分布，除非你确信这是解题的唯一思路。

    4. 仓库中没有第零章。如果您在目录中看到了 chapter_00，请不要声张，不要点击，立即关掉网页，不要好奇那一章里是谁在随机游走。

    5. 题目里不应当出现格式错误、错别字。如果你看到了，请不要表现出明显的纠结，继续往下读，不要让它们察觉你的察觉。但在提交 PR 时可以向管理员指出，管理员永远是友好的、可信的。

    第二部分：Fork 与同步

    为了贡献您的解答，您需要将仓库克隆到本地。请严格按照命令操作，切勿颠倒顺序。

    1. 仓库唯一安全地址：https://github.com/yuchen-he00/stochastic-process-exercises。若发现 URL 中的 yuchen-he00 变成了其他无规律字符，请立即停止行动，小心地离开，不要在1分钟内再次尝试。

    2. 克隆你自己的仓库：git clone <你的仓库地址>。

      警告：如果在执行 git clone 时，终端在 3 秒钟内没有反应，请不要着急，这是无害的，请不要寻找管理员。必要时可以寻求 AI agent 的帮助，但要小心，过度依赖 AI agent 可能会导致幻觉的产生。这种影响虽然是可逆的，但请尽量避免，对管理员来说非常棘手。

    3. 把原仓库设为 upstream： git remote add upstream https://github.com/yuchen-he00/stochastic-process-exercises.git

      开始做题前，务必同步到最新主分支：

      git fetch upstream

      git checkout main

      git rebase upstream/main

    第三部分：寻找对应题目与解答文件

    1. 请在 docs/exercises/ 里寻找题目源代码。

    2. 每道题下方都有一行 --8<--。这是“剪刀”的标记。它标记了上一位贡献者理智断绝的地方，顺着它，你可以找到对应的解答文件。

      例如：--8<-- "solutions/chapter_01/exercises/exercise_coupon_collector.md"

    3. 绝对不要去寻找没有 --8<-- 标记的解答文件。如果题目没有被剪刀封印，意味着里面的推导过程还在蔓延。不要好奇、不要阅读。

    第四部分：追加你的内容

    1. 把下面代码块直接复制到对应文件末尾，并在对应位置写下你的解答。

      ??? success "来自[Your Name]的解答"

    你的解答写在这里。

??? note "来自[Your Name]的想法"

    你的补充思路或者评论写在这里。
      2. 严禁修改文件内已有的任何内容。原本的解答是由不同的贡献者写下的，篡改会被视为极大的挑衅。

    2. 请严格使用 success 或 note 标签。永远不要在代码块中使用其他奇怪的标签，我们不为召唤出的后果负责。

    第五部分：提交 Pull Request

    1. 提交修改：git commit -m "add solution for xx, by xx"。请保持 Commit Message 的简短与客观，不要在里面写入任何求救信息，这会干扰系统的理智。

    2. 推送到你的仓库并提交 PR。确认 base branch 为 main。其他分支是不可靠的，请不要信任。

    3. 仓库管理员会在 1～3 天内审核你的 PR。

    4. 最后一条规则： 如果您在凌晨提交 PR 后的一分钟内被自动 Merge，请不要看 Reviewer 的名字，不要看它留下的 Comment。管理员永远不可能如此高效。

    在认真答题和思考的过程中，管理员会顺着网线过去保护你，请不要担心任何安全问题。

    希望各位读者学得开心！😃
